# Surface Detail + Experimental Accent Design

## Goal
Refine the founder-style portfolio so the core object feels more intentional and memorable without losing the restrained premium tone.

## Approaches Considered

### 1. Pure Premium Surface Upgrade
- Focus on shell material, seams, layered plates, and lighting precision
- Safest path for preserving the current brand language
- Downside: stronger finish, but limited new identity

### 2. Premium Core With Experimental Accents
- Keep the premium founder base
- Add selective technical accents around the hero core: code fragments, restrained RGB split, scan-line style distortion, and data overlays
- Use accents as a framing device rather than a full-page visual system
- Recommended because it adds memory and edge without collapsing into cyber-noise

### 3. Fully Experimental Editorial
- Push glitch, code, overlays, and graphic distortion across the hero and section transitions
- High visual impact, but more likely to fight the brand and reduce clarity

## Recommended Direction
Use approach 2.

The site should still read as precise, intelligent, and founder-led first. Experimental details should only appear where they make the system core feel more alive and more authored.

## Visual Design

### Surface Language
- Add visible shell segmentation, seams, and panel variation
- Introduce restrained tonal variance across plates so the object reads as engineered
- Add one or two more precise highlights or shadow breaks instead of broad glow

### Experimental Accents
- Add a small data overlay layer near the hero core using short system phrases or code-like fragments
- Use subtle RGB split only during motion peaks, not as a constant filter
- Add a restrained scan / interference pass tied to scroll or hero entry, not persistent noise

### Motion
- Keep the core calm at rest
- Let experimental accents pulse or drift independently from the main geometry
- Use motion to imply signal analysis rather than glitch chaos

## Constraints
- No heavy textures, video assets, or postprocessing stacks
- No full-page glitch treatment
- No readable fake code blocks large enough to distract from the copy
- Maintain current device-tier performance strategy

## Success Criteria
- The hero core feels more premium at first glance
- The experimental layer adds identity without making the page feel noisy
- The scene remains usable and elegant on desktop
- Build stays clean and console stays clean
